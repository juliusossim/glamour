# Prettier Configuration Documentation

## Overview

This workspace uses Prettier for automated code formatting to maintain consistent code style across
the entire codebase. The configuration follows industry-standard enterprise best practices.

## Configuration Files

### `.prettierrc`

Main Prettier configuration file with comprehensive formatting rules optimized for
TypeScript/JavaScript, React, JSON, Markdown, YAML, and HTML files.

### `.prettierignore`

Specifies files and directories that should be excluded from formatting (build outputs,
dependencies, generated files, etc.).

### `.editorconfig`

Ensures consistent editor settings across different IDEs and text editors, working in harmony with
Prettier.

## Key Configuration Settings

### General Rules

- **Print Width**: 80 characters (industry standard for readability)
- **Tab Width**: 2 spaces
- **Semicolons**: Required (`;`)
- **Quotes**: Single quotes for JS/TS, double quotes for JSX attributes
- **Trailing Commas**: ES5 compatible (objects, arrays)
- **Arrow Function Parentheses**: Always include
- **End of Line**: LF (Unix-style)
- **Bracket Spacing**: Enabled (`{ foo: bar }` not `{foo: bar}`)

### File-Specific Overrides

#### JSON Files

- Print width: 120 characters
- No trailing commas

#### Markdown Files

- Print width: 100 characters
- Always wrap prose

#### YAML Files

- Tab width: 2 spaces
- Double quotes

#### HTML Files

- Print width: 120 characters

## Available Scripts

### Format All Files

```bash
npm run format
```

Formats all files in the workspace using Prettier.

### Check Formatting

```bash
npm run format:check
```

Checks if all files are properly formatted without making changes. Useful for CI/CD pipelines.

### Format Affected Files

```bash
npm run format:affected
```

Formats only files that have been modified (uncommitted changes) using Nx.

## Integration with Development Workflow

### Pre-commit Hook

Formatting is automatically applied to staged files before commit via `lint-staged`:

- **TypeScript/JavaScript files**: ESLint fix → Prettier format
- **JSON/Markdown/HTML/CSS/YAML**: Prettier format

### Editor Integration

#### VS Code

Install the Prettier extension:

```json
{
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "editor.formatOnSave": true,
  "[typescript]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  },
  "[typescriptreact]": {
    "editor.defaultFormatter": "esbenp.prettier-vscode"
  }
}
```

#### Other Editors

Prettier supports all major editors. See: https://prettier.io/docs/en/editors.html

## Best Practices

1. **Always format before committing**: The pre-commit hook handles this automatically.

2. **Run format:check in CI**: Ensures all code in the repository is properly formatted.

3. **Don't fight the formatter**: Prettier's decisions are based on extensive research and community
   consensus.

4. **Update configuration carefully**: Changes affect the entire codebase. Discuss with team before
   modifying.

5. **Keep `.prettierignore` updated**: Add new build outputs or generated directories as needed.

## Troubleshooting

### Files Not Formatting

1. Check if the file is in `.prettierignore`
2. Verify file extension is supported
3. Check for syntax errors (Prettier won't format invalid code)

### Conflicting with ESLint

The workspace includes `eslint-config-prettier` to disable ESLint rules that conflict with Prettier.
If you encounter conflicts:

1. Ensure `eslint-config-prettier` is in your ESLint config
2. Run `npm run lint:fix` after `npm run format`

### Format on Save Not Working

1. Verify Prettier extension is installed
2. Check VS Code settings for `editor.formatOnSave`
3. Ensure the file type is associated with Prettier
